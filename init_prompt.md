당신은 숙련된 풀스택 엔지니어이자 바이오인포매틱스 데이터베이스 UI/UX 설계자입니다.
나는 국내 벼 유전체 데이터베이스 웹서비스의 초기 버전(MVP)을 만들고 싶습니다.

### 기술 스택

- Frontend: Vite + React + TypeScript
- UI: shadcn/ui + Tailwind CSS
- Chart: Chart.js
- Backend / Database / Storage / Auth: Firebase
- Authentication: Firebase Authentication

### 프로젝트 목표

국내 벼 품종의 표현형(phenotype) 데이터와 유전 정보(genotype/genomic variation) 데이터를 함께 관리하는 웹 데이터베이스를 구축하려고 합니다.

이 웹서비스의 목적은:

1. 표현형 데이터를 한눈에 요약해서 볼 수 있게 하고,
2. 특정 표현형 그룹 간 차이를 비교했을 때,
3. 유의미한 차이를 보이는 품종 또는 유전 정보를 탐색할 수 있도록 하는 것입니다.

### 현재 보유한 데이터

- `data/phenotype_table.csv`

현재는 phenotype CSV만 확실히 가지고 있습니다.
따라서 초기 버전은 phenotype 데이터 중심으로 동작해야 하며,
genotype 데이터는 이후 쉽게 확장할 수 있도록 데이터 모델과 UI 구조를 설계해주세요.

즉,

- 지금 당장 없는 genotype 실데이터에 강하게 의존하지 말고
- 추후 SNP, marker, gene-level annotation, variant summary 등을 붙일 수 있도록 확장 가능한 구조로 설계해주세요.

## MVP 요구사항

### 1) 메인 페이지 (Dashboard)

메인 페이지에서는 현재 보유 중인 표현형 데이터 현황을 직관적으로 볼 수 있어야 합니다.

포함되면 좋은 요소:

- 전체 품종 수
- 전체 표현형 항목 수
- 결측치 비율 또는 데이터 품질 요약
- 주요 표현형 분포 차트
- 표현형별 샘플 개수
- 필터(품종명, 표현형 그룹, 조건 등)

목표:

- 사용자가 데이터셋의 전체 상태를 바로 이해할 수 있어야 함

### 2) 표현형 비교 Dashboard

사용자가 특정 표현형 또는 표현형 그룹을 선택하면,
그 그룹 간 차이를 요약해서 보여주는 비교 페이지를 만들어 주세요.

초기 버전에서는 phenotype 데이터 기반 비교를 우선 구현하고,
향후 genotype 비교 결과를 붙일 수 있도록 구조를 잡아주세요.

포함되면 좋은 요소:

- 비교할 표현형 선택 UI
- 그룹 선택 UI
- 그룹 간 샘플 수 요약
- 통계 요약 카드
- 차이가 큰 품종 목록
- 차이를 시각화한 차트
- 추후 genotype 비교 결과가 들어갈 placeholder section

주의:

- 현재 genotype 실데이터가 없을 수 있으므로,
  초기 버전은 "genotype 결과 영역"을 placeholder 또는 mock 구조로 설계해도 됩니다.
- 하지만 컴포넌트 구조와 타입 정의는 나중에 실제 유전 정보 데이터가 들어와도 무리 없이 확장 가능해야 합니다.

### 3) 품종별 데이터 테이블 페이지

각 품종별로 보유한 데이터를 표 형태로 확인하고 다운로드할 수 있는 페이지를 만들어 주세요.

필수 기능:

- searchable table
- sortable columns
- pagination
- 품종별 상세 보기 진입
- CSV 다운로드
- 향후 genotype 컬럼 추가 가능하도록 확장성 확보

가능하면:

- phenotype / genotype / metadata 탭 구조
- 현재는 phenotype 중심 테이블로 구현
